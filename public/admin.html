🚀 Complete Netlify + Supabase Setup Guide

1
Get Gemini API Key (Free)

Go to aistudio.google.com → Sign in with Google → Get API key → Create API key. Copy it (starts with AIza...).

Free tier: gemini-1.5-pro = 2 req/min, 50 req/day  |  gemini-2.0-flash = 15 req/min, 1,500 req/day
To switch model, set GEMINI_MODEL env var to gemini-2.0-flash

2
Create Supabase Project (Free)

Go to supabase.com → New Project → Choose a region close to Thailand (Singapore). Wait ~2 mins for setup.

3
Run SQL to Create Tables

In Supabase → SQL Editor → paste and run this:

-- Chat logs table CREATE TABLE chat_logs ( id BIGSERIAL PRIMARY KEY, session_id TEXT NOT NULL DEFAULT 'anon', role TEXT NOT NULL, message TEXT, ip_hash TEXT, created_at TIMESTAMPTZ DEFAULT NOW() ); -- Config table CREATE TABLE site_config ( key TEXT PRIMARY KEY, value TEXT, updated_at TIMESTAMPTZ DEFAULT NOW() ); -- Cached answers table (instant replies, no AI call) CREATE TABLE cached_answers ( id BIGSERIAL PRIMARY KEY, question TEXT NOT NULL, keywords TEXT NOT NULL, answer TEXT NOT NULL, is_active BOOLEAN DEFAULT true, hit_count INTEGER DEFAULT 0, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); ALTER TABLE chat_logs ENABLE ROW LEVEL SECURITY; ALTER TABLE site_config ENABLE ROW LEVEL SECURITY; ALTER TABLE cached_answers ENABLE ROW LEVEL SECURITY; CREATE POLICY "service_all" ON chat_logs FOR ALL USING (true); CREATE POLICY "service_all" ON site_config FOR ALL USING (true); CREATE POLICY "service_all" ON cached_answers FOR ALL USING (true); CREATE INDEX idx_logs_session ON chat_logs(session_id); CREATE INDEX idx_logs_created ON chat_logs(created_at DESC); CREATE INDEX idx_logs_role ON chat_logs(role); CREATE INDEX idx_cache_active ON cached_answers(is_active); -- Atomic hit counter (called by chat function) CREATE OR REPLACE FUNCTION increment_cache_hit(answer_id BIGINT) RETURNS void AS $$ UPDATE cached_answers SET hit_count = hit_count + 1 WHERE id = answer_id; $$ LANGUAGE sql SECURITY DEFINER;
4
Get Supabase Keys

Supabase → Project Settings → API:
Copy Project URL (e.g. https://xxxx.supabase.co)
Copy anon / public key (safe to use in serverless functions)

5
Push to GitHub

Push the full project folder to a new GitHub repo. Make sure netlify/functions/, public/, netlify.toml are all included.

6
Connect to Netlify

netlify.com → Add new site → Import from GitHub → Select repo → Deploy

7
Set All Environment Variables

Netlify → Site → Environment variables → Add these 4 variables:

GEMINI_API_KEY = AIza... (from aistudio.google.com)
ADMIN_TOKEN = YourSecurePassword123!
SUPABASE_URL = https://jjezdajcszhivlvubzfq.supabase.co
SUPABASE_ANON_KEY = eyJ...
GEMINI_MODEL = gemini-2.0-flash (optional — default is gemini-1.5-pro)
8
Trigger Redeploy & Test

Netlify → Deploys → Trigger deploy. Your site is live at https://your-site.netlify.app.
Test the chat, then check this admin panel — you should see logs appearing in real-time.
